## Codebase Patterns
- Next.js 16.2.2 with React 19.2.4, App Router, TypeScript strict mode
- Source code in `src/` directory, app routes in `src/app/`
- ESLint configured via `eslint.config.mjs` (flat config); legacy JS files excluded from lint
- Old vanilla JS app files at root: app.js, data.js, icons.js, index.html, style.css
- Global CSS lives at `src/styles/globals.css`, imported in `src/app/layout.tsx`
- Fonts loaded via CSS `@import` from Google Fonts (Ojuju + Space Mono)
- Theme switching uses `[data-theme="dark"]` attribute with CSS variable overrides
- Types in `src/lib/types.ts`, constants/helpers in `src/lib/constants.ts`
- `getScoreLabel()`, `rockDryingFactor()`, `terrainRainFactor()`, `getSeasonProfile()` are in constants.ts
- Crag data in `src/data/crags.ts` (DEFAULT_CRAGS, REGION_CRAGS, ALL_EXPLORE_CRAGS), regions in `src/data/regions.ts`
- Explore crags default terrain to 'vertical' (source data.js doesn't specify terrain for explore crags)
- Scoring engine in `src/lib/scoring.ts`: `computeScoreV2` + helpers (`calcDryingHours`, `calcDryStreak`, `getDayWindDirection`, `calcWindShelter`)
- Scoring helpers in `src/lib/scoring.ts`: `blendedScore`, `getTodayScore`, `getWeekendScore`, `estimateRockTemp`, `frictionLabel`, `getTimeSlots`, `getDayTrend`, `wxLabel`, `dryingLabel`, `dryingClass`, `windDirLabel`, `confTag`
- Scoring uses `fc.best || fc.ecmwf` pattern â€” CH2 preferred, ECMWF fallback
- Platform detection: `getPlatform()` in `src/lib/platform.ts`, `usePlatform()` hook in `src/hooks/usePlatform.ts`
- `PlatformGate` component in `src/components/PlatformGate.tsx` for conditional rendering by platform
- `blendedScore` runs computeScoreV2 separately for each model then averages; confidence based on score diff
- `getTodayScore` and `getWeekendScore` accept (fc, crag) not (cragId) â€” store lookups happen at the caller level
- `wxLabel` (weather code to text) is in scoring.ts, ported from icons.js
- Storage adapter: `storage` singleton from `src/lib/storage.ts`; keys `bitWet_v1` (crags) and `bitWet_theme` (theme)
- All localStorage access must be guarded with `typeof window === 'undefined'` for SSR compatibility
- Zustand stores live in `src/stores/`. Use `create` from 'zustand' with typed state interface.
- Crag ID generation: `name.toLowerCase().replace(/[^a-z0-9]/g, '') + '_' + Date.now()` (matches original app.js)
- Forecast store in `src/stores/forecastStore.ts`: `forecastCache` (usuals), `exploreFcCache` (explore), `fetchForecast`, `fetchAllForecasts`, `fetchExploreForecast`
- Forecast store uses `fetchForecastApi()` from `src/lib/api.ts` â€” routes through `/api/forecast` on web, calls Open-Meteo directly on iOS
- `src/lib/api.ts` exports `fetchForecastApi()` and `fetchGeocodeApi()` â€” platform-aware API abstraction
- Capacitor iOS: `build:ios` script temporarily moves `src/app/api` out before static export (route handlers incompatible with `output: 'export'`)
- `capacitor.config.ts` at project root; `webDir: 'out'` points to Next.js static export output
- `NEXT_PUBLIC_STATIC_EXPORT=true` env var triggers `output: 'export'` in next.config.ts
- `CapacitorInit` component in layout.tsx handles StatusBar and SplashScreen on iOS
- Safe area insets applied via `env(safe-area-inset-*)` CSS in globals.css for iOS notch handling
- `ios/` and `android/` directories are gitignored and eslint-ignored (generated by Capacitor)
- Auto-refresh: `startAutoRefresh(crags)` sets 30-min interval; `stopAutoRefresh()` clears it
- UI store in `src/stores/uiStore.ts`: `activeTab`, `currentSort`, `exploreSort`, `expandedDays`, `expandedExplore`, `theme`
- Theme is applied via `document.documentElement.setAttribute('data-theme', theme)` â€” CSS uses `[data-theme="dark"]` selector
- `SortMode` type: `'weekend' | 'score' | 'name'`; `TabId` type: `'usuals' | 'explore' | 'planner'`
- `WeatherIcon` component prop is `weatherCode` (not `code`) â€” accepts WMO weather code number
- Explore tab in `src/components/ExploreTab.tsx`: uses its own inline crag display (not CragCard), with `ExploreCragRow`/`ExploreCragDetail` sub-components
- Explore forecast cache keyed by `crag.id` (stable IDs from name), forecasts lazy-loaded on first tab visit via `fetchExploreForecast`
- Planner: `PlannerForm` exports `PlannerFilters` type; `PlannerResults` runs search when filters change; batches of 6, cache-first, `sortKey = avgScore * 1000 - dist`
- React 19 eslint rule `react-hooks/set-state-in-effect` forbids setState in useEffect â€” use key-based remount pattern instead
- Modal state (open/close, editing crag) is managed in `page.tsx`, not inside CragModal â€” CragModal is a controlled component
- CragCard component: `src/components/CragCard.tsx` â€” accepts `crag`, `forecast`, `index`, `onEdit`, `onRemove` props
- CragCard day expansion controlled by `useUIStore.expandedDays` â€” detail panel is separate (DayDetail, US-017)
- Main page shell: `src/app/page.tsx` ('use client') with header, `Tabs` nav, and tab-pane switching
- `ThemeInitializer` syncs Zustand theme to `data-theme` attribute; `ThemeToggle` toggles dark/light
- API route handlers in `src/app/api/*/route.ts`; use `NextRequest`/`NextResponse` from 'next/server'
- Open-Meteo API: base `https://api.open-meteo.com/v1/forecast`, models `meteoswiss_icon_ch2` (5d) and `ecmwf_ifs` (7d), `past_days=2`, `timezone=auto`
- Open-Meteo geocoding API: `https://geocoding-api.open-meteo.com/v1/search` â€” separate base URL from forecast API
- API route response patterns: forecast returns `{ best, ecmwf }`, geocode returns `{ results: [{ lat, lon, name }] }`
- Platform-aware defaults: `getPlatform()` can be called in Zustand store initializers (non-hook); use `usePlatform()` hook in components
- UI store `currentSort` initialized via `getInitialSort()` â€” returns `'score'` (web) or `'weekend'` (iOS)
